# Genesis Prefunded Balances — Virel → LiteDAG Migration

## Summary

LiteDAG is a fork of Virel (VRL). Old VRL holders get their balances on the new chain via a 1:1 snapshot injected at genesis. Users do nothing — same seed phrase, same address, balance is there.

## Architecture

```
genesis/prefunded.go          ← SINGLE SOURCE OF TRUTH
                                 PrefundedSupply constant + PrefundedBalances map
                                 Generated by snapshot-export, committed as source

cmd/snapshot-export/main.go   ← ONE-TIME TOOL
                                 Reads old Virel LMDB, outputs genesis/prefunded.go
                                 Remove after mainnet is stable

block/reward.go               ← SUPPLY FORMULA
                                 GetSupplyAtHeight += genesis.PrefundedSupply
                                 Generalized: "genesis supply isn't just block rewards"

blockchain/blockchain.go      ← GENESIS INJECTION
                                 addGenesis() iterates genesis.PrefundedBalances
                                 Generalized: "genesis can include prefunded addresses"
                                 Runs exactly once (first boot), no-op on subsequent boots
```

### Design principles

1. **Genesis state configuration, not migration code.** The codebase doesn't know about "migration." It knows that genesis state can include prefunded addresses. This is a legitimate feature (useful for testnets, airdrops, any future chain). The migration happens to populate that data from an old chain snapshot.

2. **Single source of truth.** `genesis/prefunded.go` is the only file that contains migration-specific data. Everything else reads generalized genesis state. Change one file to change everything.

3. **No consensus contamination.** Prefunded balances are injected at genesis (height 0), before any blocks are mined. The block validation path, P2P layer, mempool, and ongoing consensus are completely untouched. The injection code in `addGenesis()` only runs when the data directory is empty.

4. **Supply invariant preserved.** `CheckSupply` (fatal assert in debug mode, CLI command) sums all state balances and compares against `GetSupplyAtHeight(h)`. The formula includes `PrefundedSupply` so the invariant holds at every height.

5. **Deterministic.** The prefunded data is compiled into the binary. Every node that boots from scratch produces identical genesis state. No external dependencies, no network calls, no race conditions.

## How it works

```
Old Virel node (synced)
       │
       │ go run ./cmd/snapshot-export -datadir /path/to/virel-data
       │
       ▼
genesis/prefunded.go   ← committed to repo
       │
       │ go build ./cmd/litedag-node
       │
       ▼
litedag-node binary (prefunded data compiled in)
       │
       │ ./litedag-node --data-dir ./mainnet-data   (first boot, empty data dir)
       │
       ▼
Genesis block created → prefunded balances injected → CheckSupply passes
```

## Taint policy

- **Blocked:** Old Virel treasury (`v139diixrpv0ftmip4mgpuy92u51iq4pnmgjsfn`). Excluded from snapshot by `address.IsBlocked()`.
- **Not blocked:** Downstream recipients of treasury funds. In an account-based model, funds mix in accounts — there's no clean way to separate "tainted" from "clean" balance without deep chain analysis and arbitrary cutoffs. The founder already dumped (sold to exchanges), so 1st-hop addresses are mostly exchange hot wallets. Blocking those would punish innocent users.
- **Burn address:** Excluded (zero-value address, internal accounting).
- **Delegate addresses:** Excluded (internal staking accounting, not user balances).

## Snapshot extraction

Run on any machine with access to a synced old Virel node's data directory:

```bash
cd ~/litedag/litedag-blockchain

# First: check what height the chain DB is at
go run ./cmd/snapshot-export -datadir /path/to/virel-data
#   chain is at height 123456 — use: -height 123456

# Then: export at that height (height is required, acts as safety assertion)
go run ./cmd/snapshot-export -datadir /path/to/virel-data -height 123456
```

The `-height` flag is required. It asserts the chain DB is at exactly that height and records it in the generated output. If the height doesn't match, the tool tells you the actual height and suggests what to use.

The tool reads:
1. **Liquid balances** from the "state" index (what wallet shows)
2. **Staked funds** from the "delegate" index (funds locked in PoS delegates)

Both are credited to the owner as liquid LDG on the new chain. Stakers don't lose their staked VRL in migration.

The tool filters out: blocked addresses (treasury), burn address, delegate internal addresses, zero balances. Output is sorted by address string for deterministic, diffable results.

After running, commit the generated `genesis/prefunded.go` and rebuild the node binary.

## Branch strategy

This work lives on the `genesis` branch until the snapshot data is populated and verified.

**Merge to master** when:
- [ ] Old Virel node synced and snapshot extracted
- [ ] `genesis/prefunded.go` populated with real data
- [ ] Node builds and boots with prefunded balances
- [ ] `CheckSupply` passes in debug mode
- [ ] Known old VRL address verified via RPC `get_address`
- [ ] Old treasury verified excluded (zero balance, blocked from sending)

## TTL and cleanup plan

After mainnet launch and stability (target: **90 days post-launch**):

### What to delete
- `cmd/snapshot-export/` — the extraction tool (one-time use, done)
- `genesis/prefunded.go` — replace with a simple file:
  ```go
  package genesis
  // Virel migration balances were injected at genesis on YYYY-MM-DD.
  // Original snapshot: <N> addresses, <X> LDG total.
  // This data was removed after mainnet stabilized (90 days post-launch).
  const PrefundedSupply uint64 = <final_value>
  var PrefundedBalances = map[address.Address]uint64{}
  ```
- `GENESIS_MIGRATION.md` — this file (move key facts to CLAUDE.md history if needed)

### What stays permanently
- `genesis.PrefundedSupply` as a plain constant (supply formula needs it forever)
- The prefunded balances loop in `addGenesis()` (no-op with empty map, useful for testnets)
- `genesis.PrefundedBalances` as an empty map (the loop iterates it harmlessly)

### One commit to clean up
```bash
git checkout master
# Replace genesis/prefunded.go with constant-only version
# Delete cmd/snapshot-export/
# Delete GENESIS_MIGRATION.md
git add -A
git commit -m "chore: remove migration data, keep genesis prefunded infrastructure"
```
